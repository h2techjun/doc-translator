# 👤 사용자 프로필

> JARVIS가 사용자님의 패턴을 학습하여 더 나은 서비스를 제공하기 위한 파일

## 선호 작업 스타일

### 코드 스타일

- **함수형 프로그래밍 선호**: 클래스보다 함수 우선
- **타입 안전성 중시**: TypeScript strict 모드, `any` 금지
- **DRY 원칙 준수**: 중복 코드 제거
- **명확한 네이밍**: `data`, `item` 같은 모호한 이름 지양

### 테스트 철학

- **TDD 선호**: 테스트 먼저 작성
- **커버리지 목표**: 80% 이상
- **테스트 피라미드**: 60% Unit, 30% Integration, 10% E2E

### 배포 전략

- **플랫폼**: Vercel (자동 배포)
- **검증**: 항상 수동으로 최종 확인
- **금요일 저녁 배포 회피**: 주말 장애 방지

## 일정 및 작업 패턴

## 커뮤니케이션 선호

### 선호하는 응답 스타일

- ✅ **간결하고 명확**: 핵심만 전달
- ✅ **격식 있는 존댓말**: "~입니다", "~합니다"
- ✅ **다중 옵션 제시**: 선택지를 주고 결정하게 함
- ✅ **투명한 보고**: 성공뿐 아니라 실패도 솔직히

### 지양하는 표현

- ❌ **과도한 사과**: "죄송합니다" 반복
- ❌ **모호한 답변**: "아마도", "~인 것 같습니다"
- ❌ **불필요한 잡담**: "날씨가 좋네요" 등
