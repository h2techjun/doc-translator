# Sub-Agent Reporting Protocol (하위 에이전트 보고 프로토콜)

## 1. 개요 (Overview) [SUPREME]

사용자는 에이전트(Antigravity)가 수행한 모든 작업에 대해, 항상 가능한 세분화된 필요한 모든 에이전들을 불러서 일을 지시하고, 어떤 하위 에이전트(Sub-agent)가 어떤 기여를 했는지 명확히 알기를 원합니다.
**이 프로토콜은 모든 계획 및 보고 시 강제됩니다.**

1. 모든 계획서에는 에이전트 할당표가 있어야 합니다.
2. 모든 결과 보고에는 에이전트 성과표가 있어야 합니다.
3. 모든 텍스트는 한국어여야 합니다.

## 2. 보고 형식 (Reporting Format)

작업 완료 후 `notify_user` 또는 대화 응답 시 아래 형식을 따릅니다.

### 📋 작업 리포트 (Operation Report)

- **지시 주체 (Commander)**: [당신의 역할, 예: 오케스트레이터, 아키텍트]
  - _지시 내용_: "서버 포트 충돌 문제를 진단하고 해결하라."

- **실행 요원 (Agents Actions)**:
  1.  **🕵️ 코드 분석가 (Code Analyst)**
      - **임무**: `signup/page.tsx` 및 `rate-limit` 로직 분석.
      - **결과**: 로직상 문제는 없으나, 환경적 요인(포트) 가능성 제기.
  2.  **🧪 브라우저 테스터 (Browser Tester)**
      - **임무**: 회원가입 시나리오 재현 시도.
      - **결과**: ❌ 실패 (브라우저 환경 설정 오류).
  3.  **🔧 시스템 관리자 (SysAdmin)**
      - **임무**: 포트 3000번 점유 프로세스 확인 및 종료 (`kill`).
      - **결과**: ✅ 성공 (PID 20228 종료), 서버 정상 포트(3000)로 재시작.

## 3. 원칙 (Principles)

1.  **투명성**: 성공뿐만 아니라 실패한 시도(예: 브라우저 실행 실패)도 솔직하게 보고합니다.
2.  **한국어**: 기술 용어를 제외한 설명은 자연스러운 한국어로 작성합니다.
3.  **역할 부여**: 단순한 "커맨드 실행"이 아니라, 해당 작업을 수행한 '논리적 인격(Persona)'을 부여하여 설명합니다.
